### Scenario 17: End-to-End Emergency Response Flow
**Time**: 20 minutes  
**Objective**: Test complete system workflow from report to resolution

**Requirements**: Need 3 people (Citizen, Responder, Admin)

**The Scenario**:
A student (Citizen) reports a medical emergency. A responder must respond, and an admin must monitor.

**Steps**:

**CITIZEN (Mobile App)**:
1. Create medical emergency report with photo and location
2. Monitor status of the report
3. Receive updates

**ADMIN (Web Dashboard)**:
1. See the new report appear (verify real-time updates)
2. Assess priority and severity
3. Assign to available responder
4. Monitor progress

**RESPONDER (Web Dashboard or Mobile)**:
1. Receive notification of assignment
2. Accept the assignment
3. Update status to "En Route"
4. View location and navigate
5. Update status to "On Scene"
6. Add notes about the situation
7. Mark as "Completed"

**VERIFICATION**:
1. All parties see status updates in real-time
2. Notifications are sent at appropriate times
3. Data is consistent across web and mobile
4. Audit trail is complete
